## 🌟 О проекте

**Единый информационный веб-ресурс** для двух дачных посёлков — **ДПК «Заповедное»** и **ДПК «Колосок»**.

**Ключевые особенности:**
- Два логических подсайта: `/zapovednoe` и `/kolosok`.
- Автоматический редирект с `/` на `/zapovednoe`.
- Мгновенное переключение между посёлками (без перезагрузки страницы).
- Разделы: Главная, О посёлке, Участки, Инфраструктура, Галерея, Документы, Контакты, Новости.
- Полноценная **админ-панель (CMS)** с разграничением прав:
  - **Администратор** — полный контроль над всем.
  - **Председатель** — управление только своим посёлком.
  - **Житель** — после входа видит документы своего посёлка.
  - **Гость** — просмотр общедоступного контента.
- Адаптивный дизайн (ПК, планшеты, смартфоны).
- SEO-готовность (мета-теги, robots.txt, sitemap.xml).

## 🚀 Установка и запуск (локально)

### 4.1 Клонирование репозитория

```bash
git clone https://github.com/ваш-логин/dpk-site.git
cd dpk-site
```

Если у вас нет Git, скачайте ZIP-архив с GitHub и распакуйте.

### 4.2 Запуск бэкенда

Откройте **первый терминал** (или командную строку) и выполните:

```bash
cd backend
npm install
npm run dev
```

**Что произойдёт:**
- Установятся все зависимости (Express, SQLite и др.)
- Создастся файл `database.sqlite` в папке `backend`.
- Создадутся таблицы: `users`, `news`, `documents`, `plots`, `infrastructure`, `gallery`, `contacts`.
- Добавятся три тестовых пользователя (см. раздел 4.4).
- Сервер начнёт слушать порт 5000.

Вы увидите сообщение:
```
Сервер запущен на http://localhost:5000
```

**Если возникла ошибка `SQLITE_ERROR: no such table`** – удалите `database.sqlite` и перезапустите.

### 4.3 Запуск фронтенда

Откройте **второй терминал** (не закрывая первый) и выполните:

```bash
cd frontend
npm install
npm install react-helmet-async   # если пакет не был установлен автоматически
npm start
```

- Автоматически откроется браузер с адресом `http://localhost:3000`.
- Вы будете перенаправлены на `http://localhost:3000/zapovednoe`.

### 4.4 Тестовые учётные записи

## 🧠 Возможности сайта и управление

### 7.1 Для гостей (неавторизованных)

- **Главная** – краткое описание посёлка, 3 последние новости, 4 последних фото.
- **О посёлке** – информация из бэкенда (адрес, телефон, описание, устав).
- **Участки** – каталог с фильтрацией по площади, цене, статусу. Внизу – карта (iframe Яндекс.Карт).
- **Инфраструктура** – карточки с иконками (например, ⚡ электричество, 💧 вода).
- **Галерея** – сетка фото и видео.
- **Новости** – список всех новостей, клик по заголовку открывает детальную страницу.
- **Контакты** – адрес, карта проезда, форма обратной связи (отправляет сообщение в БД).

### 7.2 Для жителей (авторизованных)

- **Документы** – раздел становится доступным. Житель видит только документы, загруженные для **его** посёлка.
- Всё остальное – как у гостя.

### 7.3 Для председателя

- Доступ к админ-панели через кнопку «Админка» в шапке.
- Возможность **добавлять, редактировать, удалять**:
  - Новости (только своего посёлка)
  - Документы (только своего)
  - Участки (только своего)
  - Объекты инфраструктуры (только своего)
  - Фото и видео в галерею (только своего)
- Просмотр сообщений из формы обратной связи (только для своего посёлка).
- Управление пользователями своего посёлка: может изменить роль **жителя** на **гостя** и наоборот.

### 7.4 Для администратора

- Полный доступ ко **всем данным** обоих посёлков.
- Может назначать любые роли (`admin`, `chairman`, `resident`, `guest`) любым пользователям.
- Видит все сообщения из формы обратной связи (всех посёлков).

### 7.5 Переключение между посёлками

- В правой части шапки – две кнопки: **«Заповедное»** и **«Колосок»**.
- При клике меняется URL и весь контент (главная, участки, документы и т.д.) обновляется без перезагрузки страницы.

## 🩺 Устранение неполадок (FAQ)

| Проблема | Вероятная причина | Решение |
|----------|-------------------|---------|
| **Ошибка `SQLITE_ERROR: no such table`** | База данных не создалась или таблицы отсутствуют. | Удалите `backend/database.sqlite` и перезапустите бэкенд. |
| **Ошибка CORS в браузере** | Фронтенд (порт 3000) пытается обратиться к бэкенду (порт 5000), но CORS не разрешает. | Убедитесь, что в `server.js` подключён `cors()`. Также проверьте, что бэкенд запущен. |
| **Файлы не загружаются (uploads)** | Отсутствует папка `uploads/` или нет прав на запись. | Создайте папку `backend/uploads/` вручную и дайте права `chmod 755`. |
| **React не запускается: `Module not found: react-helmet-async`** | Пакет не установлен. | Выполните `npm install react-helmet-async` в папке `frontend`. |
| **При переключении посёлка тема не меняется** | В `Header.js` не используется `themeClass`. | Проверьте, что компонент Header получает `settlement` и добавляет класс `<header className={themeClass}>`. |
| **Документы недоступны даже после входа** | Возможно, у пользователя роль `guest` или он не из того посёлка. | Зайдите в админ-панель и проверьте/измените роль. |
| **Админ-панель пустая или нет кнопок** | Не установлены компоненты админки или не настроены маршруты. | Убедитесь, что в `App.js` есть `<Route path="/admin" element={...}>` и все дочерние маршруты. |
| **При отправке формы обратной связи ошибка 500** | Бэкенд не может сохранить запись в таблицу `contacts`. | Проверьте, что таблица `contacts` создана (посмотрите `database.js`). Добавьте её, если отсутствует. |
| **Карта участков не отображается** | Используется iframe с заглушкой. | Получите API-ключ Яндекс.Карт и замените src на реальный конструктор карты с отметками участков. |
| **Не удаётся войти с тестовыми учётными данными** | Пользователи не были созданы. | При первом запуске бэкенда они добавляются автоматически. Если нет – вставьте SQL-запрос вручную через `sqlite3`. |

## 🌍 Деплой на хостинг

### 11.1 Деплой бэкенда на Render.com

1. Зарегистрируйтесь на [render.com](https://render.com).
2. Нажмите **New +** → **Web Service**.
3. Подключите ваш GitHub-репозиторий.
4. Настройте:
   - **Name**: `dpk-backend`
   - **Root Directory**: `backend`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
5. В разделе **Environment Variables** добавьте:
   - `JWT_SECRET` = любая длинная строка
   - `PORT` = `5000` (или оставить пустым, Render задаст свой)
6. Нажмите **Create Web Service**.
7. После деплоя вы получите URL вида `https://dpk-backend.onrender.com`.

### 11.2 Деплой фронтенда на Netlify

1. Зарегистрируйтесь на [netlify.com](https://netlify.com).
2. Нажмите **Add new site** → **Import an existing project**.
3. Подключите GitHub и выберите репозиторий.
4. Настройте:
   - **Base directory**: `frontend`
   - **Build command**: `npm run build`
   - **Publish directory**: `frontend/build`
5. В разделе **Environment variables** создайте переменную:
   - `REACT_APP_API_URL` = URL вашего бэкенда на Render (например, `https://dpk-backend.onrender.com/api`)
   - **Важно:** после этого в `frontend/src/services/api.js` нужно использовать `process.env.REACT_APP_API_URL`.
6. Нажмите **Deploy site**.

### 11.3 Настройка домена и SSL

- На Render: перейдите в настройки вашего Web Service → **Custom Domain** → добавьте домен (например, `api.dpk-site.ru`). SSL выдаётся автоматически.
- На Netlify: **Domain settings** → **Add custom domain** → укажите ваш домен (например, `dpk-site.ru`). Netlify также автоматически предоставляет SSL-сертификат через Let's Encrypt.

**Пример файла `api.js` для продакшена:**
```javascript
const api = axios.create({
  baseURL: process.env.REACT_APP_API_URL || 'http://localhost:5000/api',
});
```
